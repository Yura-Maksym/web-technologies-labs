# 🍫 Confectionery REST API & Web App

Повноцінний вебзастосунок для каталогу кондитерської продукції. Проєкт об'єднує результати 4-х лабораторних робіт і включає REST API бекенд, клієнтську частину (фронтенд) з авторизацією, покриття автоматичними тестами та налаштований CI/CD пайплайн.

## 🛠 Технологічний стек
* **Бекенд:** Python 3.11, FastAPI, SQLAlchemy (база даних SQLite)
* **Фронтенд:** HTML5, Vanilla JS (Fetch API), Bootstrap 5 (UI)
* **Збереження стану:** `localStorage` (Shared State для JWT-токенів)
* **Тестування:** Pytest (Unit-тести), Selenium (Web Scraping / E2E)
* **DevOps / Деплой:** Docker, GitHub Actions (CI), Render (CD)

## 🚀 Основний функціонал
1. **REST API:** Повноцінний CRUD для сутностей каталогу (Продукти, Категорії).
2. **Клієнтський інтерфейс:** Три зв'язані сторінки (Вхід, Каталог, Профіль).
3. **Авторизація:** Захист маршрутів, генерація та перевірка токенів на стороні клієнта.
4. **CI/CD Автоматизація:** Конвеєр автоматично перевіряє код тестами та розгортає нові версії з гілки `main` у хмару.

## 📂 Структура проєкту
```text
├── .github/workflows/   # Налаштування CI/CD для GitHub Actions
├── frontend/            # Клієнтська частина (HTML, JS)
│   ├── index.html       # Сторінка авторизації
│   ├── catalog.html     # Каталог товарів із динамічним списком
│   └── profile.html     # Особистий кабінет із захищеним токеном
├── database.py          # Налаштування підключення до SQLite
├── models.py            # ORM моделі бази даних (SQLAlchemy)
├── schemas.py           # Pydantic схеми для серіалізації та валідації
├── main.py              # Головний файл FastAPI (маршрути та логіка)
├── test_main.py         # Набір Unit-тестів (Pytest)
├── scraper.py           # Скрипт для веб-скрапінгу (Selenium)
├── Dockerfile           # Інструкції для збірки ізольованого контейнера
└── requirements.txt     # Залежності Python
```

## 🔗 Основні API Ендпоінти
| Метод | Маршрут | Опис |
|---|---|---|
| `POST` | `/login` | Авторизація користувача (admin/1234) |
| `GET` | `/categories/` | Отримання списку всіх категорій |
| `POST` | `/categories/` | Створення нової категорії |
| `GET` | `/products/` | Отримання списку всіх цукерок |
| `POST` | `/products/` | Додавання нової цукерки з прив'язкою до категорії |

## 🌐 Живий демо-доступ
Застосунок розгорнуто на хмарній платформі Render.
**Посилання:** https://web-technologies-labs-wn6g.onrender.com/
*(Для входу використовуйте логін `admin` та пароль `1234`)*

> **Примітка:** Оскільки використовується безкоштовний тариф хостингу, при тривалій неактивності сервер переходить у сплячий режим. Перше завантаження сторінки може тривати до 50 секунд. Наступні запити оброблятимуться миттєво.

## 💻 Локальний запуск та тестування
**Запуск сервера:**
1. Встановіть необхідні залежності: `pip install -r requirements.txt`
2. Запустіть сервер: `uvicorn main:app --reload`
3. Відкрийте технічну документацію Swagger UI: `http://127.0.0.1:8000/docs`

**Запуск автоматичних тестів:**
Для перевірки працездатності API виконайте команду в терміналі:
`pytest test_main.py -v`
